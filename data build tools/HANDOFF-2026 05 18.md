- **MapLibre GL JS v5.5.0** — globe projection, `dragRotate: false`, `maxPitch: 0` (prevents deck.gl overlay misalignment)

## Data layout

```
data/index.json              — 11,898 eclipse index records (loaded at startup)
data/besselian/*.json        — 50 century chunks, uncompressed JSON arrays
data/paths/*.json.gz         — 50 century chunks, gzipped JSON dicts keyed by cat_no string
data/basemap/*.geojson.gz    — Natural Earth basemap layers
data/basemap/cities.geojson.gz — cities database (not yet used in search)
```

### index.json record fields
`year month day td_ge eclipse_type etype gamma magnitude lat_dd_ge lng_dd_ge sun_alt sun_azm path_width central_duration duration_secs saros luna_num cat_no julian_date t0 nSeq nSer _chunk`

### besselian chunk record fields
`year month day td_ge dt dt_source luna_num saros eclipse_type gamma magnitude lat_ge lng_ge lat_dd_ge lng_dd_ge sun_alt sun_azm path_width central_duration duration_secs cat_no canon_plate julian_date t0 x0..x3 y0..y3 d0..d2 mu0..mu2 l10..l12 l20..l22 tan_f1 tan_f2 tmin tmax etype PNS UNS NCN nSer nSeq nJLE`

`dt` = ΔT in seconds. `dt_source` = one of: `"USNO observed"`, `"USNO observed (nearby)"`, `"USNO predicted"`, `"Espenak-Meeus"`, `"SMH2016 LOD extrapolation"`, `"SMH2016 LOD (ancient)"`

### path chunk record fields
`cat_no year month day type ge centreline umbra_n umbra_s umbra_ovals penumbra_n penumbra_s terminator_first terminator_last bisector terminator_first_n_idx terminator_first_s_idx terminator_last_n_idx terminator_last_s_idx`

All coordinates `[lon, lat]`. Longitudes may be unwrapped (outside ±180) for antimeridian continuity — this is intentional, deck.gl handles it correctly.

**umbra_ovals**: closed rings (first point == last point), CW winding. Strip the closing vertex before passing to `SolidPolygonLayer`. Use `depthWriteEnabled: false` on the corridor `SolidPolygonLayer` so ovals render above it without z-fighting holes.

## Key architectural decisions (do not change without good reason)

1. **deck.gl `interleaved: false`** — interleaved causes z-fighting shimmer on the globe. Non-interleaved means the overlay is a flat screen-space layer, so `dragRotate: false` is essential to prevent the globe spinning out from under the paths.

2. **GeoJSON is gone** — all eclipse geometry is deck.gl. MapLibre is basemap only. No `addSource`/`addLayer`/`removeSource`/`removeLayer` for eclipse data.

3. **Bisector lines are hidden** — the bisector data has rendering issues (some centuries have interleaved branch data). Removed from `penPaths` array. Do not re-add without investigating the data.

4. **Penumbra fill is removed** — the closed-ring assembly was too complex and unreliable. Penumbra boundaries are shown as lines only.

5. **`SolidPolygonLayer` corridor uses `depthWriteEnabled: false`** so the umbra ovals (also `SolidPolygonLayer`) can render above it correctly.

6. **`alt:` search syntax removed** — observer altitude is fetched automatically via Open-Elevation API and stored in `_lookedUpAlt`. Never written to the search field.

7. **Scan is automatic** — debounced 800ms after typing stops when coords are present. No scan button. Cancel button remains.

## search_parser.js filter object

```js
{
  years:     { min, max, exactPair?, todayMonth?, todayDay?, todayYear? } | null,
  months:    [1..12] | null,
  days:      [1..31] | null,
  saros:     number | null,
  types:     ['total'|'annular'|'hybrid'|'partial'] | null,
  obscRange: { min, max } | null,
  coords:    { lat, lon } | null,
  text:      string | null,
  today:     bool
}
```

Supported search tokens: `today+`, year (with BCE/BC/AD/CE suffixes), month name, day, `saros N` or `sN`, type (full or 2-char prefix), `>N%` / `N+` / `<N%` / `N-` obscuration, `(lat, lon)` coordinates (many formats).

`exactPair: true` means a single bare positive year — matches both CE year and BCE equivalent (e.g. `974` matches year 974 and year -973).

Type matching uses first character only: `eclipse_type "Tm"` → classified as total.

## Rendering order in drawEclipsePath (important)

1. Penumbra boundary lines (PathLayer)
2. **Umbra ovals** (SolidPolygonLayer, `depthWriteEnabled: false`)  ← must be before corridor
3. **Umbra corridor fill** (SolidPolygonLayer, `depthWriteEnabled: false`) ← writes depth after ovals
4. Umbra boundary lines (PathLayer)
5. Centreline (PathLayer, red)
6. Penumbra lines / terminator lemniscates (PathLayer)
7. GE point — white halo + red dot (two SolidPolygonLayers, `depthWriteEnabled: false`)

## ΔT display

Global circumstances always shows ΔT. When besselian rec is loaded (after computeLocal), uses `rec.dt` and `rec.dt_source`. Before that, falls back to inline JS port of `delta_t.py` (same formula). The fallback functions are `formulaDt(year)` and `dtSourceFallback(year)` in index.html — they exist purely as fallback and should not be called when rec is available.

## Known issues / deferred features

### Not yet done from the plan:
- **Fresh start**: on app load, auto-select next total eclipse at current location, show on Map tab
- **Permanent blue dot** at current location on map (even without eclipse selected)
- **PWA / service worker** — high priority, unlocks offline install on iPhone
- **Share link** — URL hash already encodes state, just needs a share button
- **KMZ download** — geometry available, needs JSZip
- **Lat/lon/alt editable** in local circumstances panel
- **Compass/AR direction** to eclipse from current position
- **City name search** — `data/basemap/cities.geojson.gz` exists but not wired to search
- **Intelligent list scrolling** — scroll selected eclipse into view
- **Toggle offline basemap** for testing (Settings toggle)
- **Cloud cover overlay** — ERA5 climatology by day-of-year, ~90MB offline, not built yet
- **Topographic shadow overlay** — ShadeMap API (free key from shademap.app/about), pre-computed viewshed for low-sun eclipses (sun_alt < 20°)

### Known rendering issue:
- Umbra ovals still have minor hole artifacts on some eclipses when zoomed out. The root cause is two `SolidPolygonLayer` instances sharing the depth buffer. `depthWriteEnabled: false` on both layers mitigates it but doesn't fully eliminate it. This is a known deck.gl limitation in non-interleaved mode. Acceptable for now.

### Data notes:
- `bisector` field exists in data but is hidden in the UI (some centuries have interleaved branch corruption)
- Join indices (`terminator_*_n_idx`) are null for ~30% of total/annular eclipses — these are single-limit (polar) eclipses where only one penumbra boundary exists. This is correct, not a bug.
- `update_dt.py` in `data build tools/` should be run annually to update ΔT values with fresh USNO observed data, then regenerate path chunks

## Workflow notes

- Claude cannot push to GitHub directly. Claude edits a local clone, then presents output files for download.
- Always `git pull` at the start of a session to get the latest pushed state.
- The user pushes all changed files together — no selective file management needed.
- Build timestamp is visible in Settings → About. Update it when shipping index.html.
- Always validate inline JS with `node --check` before shipping.
- The user's preferences: be concise, don't guess, don't throw away working solutions, don't break things that work, remember decisions already made.
