## Tab structure (as of this session)

Four tabs, in this order:

| Tab | id | Default |
|---|---|---|
| Map | `tab-map` | **Active on load** |
| Eclipse | `tab-eclipse` | Shows selected eclipse details |
| Search | `tab-search` | Search + filter list |
| Settings | `tab-settings` | About (open) / Instructions / Settings |

**Map is the default active tab.** `initMap()` is called explicitly at startup (not via `switchTab`) because the map tab starts active. See startup sequence below.

## Tab state management

Simple and clean — do not complicate it:

- `switchTab(name)` — switches tabs, calls `onMapTabActivated()` for map tab
- `updateEclipseTabState()` — shows `data-panel` or `eclipse-placeholder` based on `selectedEntry`
- Selecting an eclipse calls `switchTab('eclipse')` then `computeLocal()`
- `clearSelection()` calls `updateEclipseTabState()`

---

## Startup sequence

1. `buildTzSelect()` — populate timezone selector
2. `initMap()` — called immediately since map is default tab
3. `fetch(index.json)` — loads catalogue
4. On load: find next total/annular eclipse, set `selectedEntry`, call `computeLocal()`, call `updateMapState()` if `mapReady`
5. `style.load` event → sets `mapReady = true` → calls `updateMapState()` → draws path if `selectedEntry` set

The two async operations (index fetch, map style load) can complete in either order. Both paths lead to `updateMapState()` being called with `selectedEntry` set.

## Map status overlay

`map-status` element (pill at bottom-centre of map) has two layers:

- **Persistent**: selected eclipse name (set via `updateHeaderSelection()` → `_renderMapStatus()`)
- **Transient**: loading/error messages (set via `setMapStatus(msg)`)

Transient overrides persistent. `setMapStatus(null)` clears transient and restores persistent.

## Eclipse list — icon + date + duration

Three columns: `2rem 1fr 4.5rem` (icon, date, duration). No header row. No magnitude column.

Type icons are inline SVGs from `typeIcon(tc)`:
- **Total** — white blurred corona (feGaussianBlur stdDeviation=4), padded viewBox so glow isn't clipped
- **Annular** — thin orange ring
- **Hybrid** — full glow unclipped, black semicircle left, orange ring right
- **Partial** — grey crescent

## Obscuration vs magnitude

These are different. Magnitude = fraction of solar diameter covered. Obscuration = fraction of solar area covered. The filter uses obscuration, computed from magnitude via:

```js
osc = (Math.acos(1 - 2*mag) - Math.sin(Math.acos(1 - 2*mag))) / Math.PI * 100
```

Do NOT round this value before comparison (that was a bug, now fixed in search_parser.js).
